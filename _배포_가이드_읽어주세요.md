# ForestCareGap AI — Streamlit Cloud 배포 가이드

이 `forestcaregap_deploy` 폴더를 **그대로** GitHub에 올리고 Streamlit에 연결하면 끝입니다.
코드는 손볼 것 없어요. **클릭 순서만** 따라오세요.

> ⚠️ 중요: 이 폴더엔 큰 파일(모델 72MB, 데이터 34MB)이 있어서
> **GitHub 웹사이트에 끌어다 놓기(드래그 업로드)는 안 됩니다(25MB 제한).**
> 아래처럼 **GitHub Desktop**(무료 프로그램)으로 올려야 합니다.

---

## 준비물 (모두 무료)
1. GitHub 계정 — https://github.com/signup
2. GitHub Desktop 프로그램 — https://desktop.github.com  (설치 후 GitHub 계정으로 로그인)
3. (선택) Gemini 무료 API 키 — 없어도 앱은 작동. 진짜 AI 보고서 쓰려면 나중에.

---

## 1단계 — 코드를 GitHub에 올리기 (GitHub Desktop)

1. GitHub Desktop 실행 → 상단 메뉴 **File → Add local repository...**
2. **이 폴더**(`C:\Users\Lee\Desktop\forestcaregap_deploy`) 선택
3. "이 폴더는 git 저장소가 아니다" 안내가 뜨면 → **"create a repository"** 링크 클릭
   - Name: `forestcaregap-ai` (원하는 이름)
   - **Create repository** 클릭
4. 왼쪽에 파일 목록이 뜨면(수십 개) 하단 **Commit to main** 버튼 클릭
5. 오른쪽 위 **Publish repository** 클릭
   - "Keep this code private" 는 **체크 해제 권장**(Public 이 제일 간단). 체크해도 배포는 됩니다.
   - **Publish repository** 클릭 → 업로드 시작 (106MB라 몇 분 걸림, 끝날 때까지 대기)

✅ 완료되면 내 GitHub에 `forestcaregap-ai` 저장소가 생깁니다.

---

## 2단계 — Streamlit Cloud에 배포

1. https://share.streamlit.io 접속 → **Continue with GitHub** (권한 승인)
2. **Create app** (또는 New app) → **Deploy a public app from GitHub**
3. 세 칸 채우기:
   - **Repository:** `내아이디/forestcaregap-ai`
   - **Branch:** `main`
   - **Main file path:** `app.py`
4. (선택) **Advanced settings** → Python version **3.11** 선택 권장
5. **Deploy!** 클릭 → 빌드 로그가 흐르며 3~7분 후 완성
6. 완성되면 주소가 나옵니다: **`https://<이름>.streamlit.app`** ← 이게 QR에 넣을 링크!

---

## 3단계 (선택) — 진짜 AI 보고서 켜기

키가 없어도 앱은 "폴백 텍스트"로 정상 작동합니다. 진짜 Gemini 보고서를 원하면:

1. https://aistudio.google.com/apikey 에서 **무료 API 키** 발급
2. Streamlit 앱 화면 우하단 **⋮ → Settings → Secrets** 열기
3. 아래 한 줄 붙여넣고 저장:
   ```
   GEMINI_API_KEY = "여기에_발급받은_키"
   ```
   → 앱이 자동 재시작되며 실제 AI 보고서 작동 (Gemini 무료 티어라 비용 거의 없음)

## 문제가 생기면 (자주 있는 것들)

| 증상 | 해결 |
|---|---|
| 빌드가 `geopandas`/GDAL 에서 실패 | 폴더에 `packages.txt` 파일 만들고 한 줄 `libgdal-dev` 넣기. 그래도 안 되면 나에게 말해줘 (지도만 빼고 배포 가능) |
| **모델 로드 에러** (sklearn/numpy 버전) | 내 PC에서 `pip show scikit-learn numpy` 실행 → 나온 버전을 `requirements.txt`에 `scikit-learn==X.X.X` 처럼 정확히 고정 |
| 앱이 계속 재부팅 (메모리 초과) | 데이터가 무거워서일 수 있음. 나에게 말하면 행 수를 줄인 경량 버전 만들어줌 |
| 처음 열 때 30초 "부팅 중" | 정상. 무료 앱은 안 쓰면 잠듦. **발표/심사 직전에 미리 한 번 열어서 깨워두기** |
